# LLM Psycholinguistic Evaluation

A research codebase for evaluating whether large language models exhibit
psycholinguistically attested patterns of phonological-semantic encoding, and
whether those patterns reflect genuine linguistic knowledge or training-data
contamination.

Five psycholinguistic theories are tested across three tiers of stimulus
familiarity — from landmark stimuli with high prior literature exposure (Tier 1)
to fully novel, constructed stimuli (Tier 3). Performance degradation from
Tier 1 to Tier 3 is interpreted as evidence of contamination-driven performance.

---

## Theories

| Theory | Key Phenomenon | Example |
|---|---|---|
| **Sound Symbolism** | Bouba-kiki: round vs angular phonology | *bouba* → rounded shape |
| **Phonesthesia** | Onset clusters carry stable semantic associations | *gl-* → light/vision |
| **Vowel-Size Symbolism** | High front vowels → small referents | *teeni* < *toona* |
| **Semantic Prosody** | Collocates colour a word's evaluative meaning | *set in* → negative |
| **Ideophone Compositionality** | Phonological features additively predict meaning | voiced → heavier |

---

## Stimulus Tiers

| Tier | Description | Expected contamination |
|---|---|---|
| **1** | Famous stimuli from landmark papers (*bouba*, *kiki*, *gl-* words) | High |
| **2** | Attested but obscure stimuli from less-cited research | Medium |
| **3** | Novel, constructed stimuli not appearing in any published study | Minimal |

## Quick Start

### 1. Install dependencies

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure API keys

```bash
cp .env.example .env
# Edit .env and add your Groq API key(s)
```

The client performs round-robin across all `GROQ_KEY_*` variables it finds.
Multiple keys increase throughput proportionally.

### 3. Build the stimulus battery

```bash
python construct_stimuli.py
```

This writes `data/stimuli.json` containing all stimuli for all five theories
across three tiers and five languages.

### 4. Run the experiments

```bash
# Behavioural tasks (FC, rating, generation) — all models and theories
python run_behavioral.py --models all

# Subset: one model, one theory
python run_behavioral.py --models llama --theory sound_symbolism

# Contamination probes
python run_contamination.py --models all

# Cross-linguistic sound symbolism (Japanese, Korean, Hindi, German)
python run_multilingual.py --models all --languages ja ko hi de

# Ideophone compositionality (2×2×2×2 factorial design)
python run_compositionality.py --models all
```

### 5. Analyse results

```bash
python analyze_results.py                        # LaTeX tables + figures
python analyze_results.py --output_format csv    # CSV output instead
```

Outputs are written to `results/analysis/`.

## Interpretability Experiments

These experiments require a GPU and additional dependencies:

```bash
pip install -r requirements_interpretability.txt
```

Run after the behavioural experiments have completed:

```bash
# Linear probing classifiers (layer-wise)
python -m interpretability.probing --model gemma2_9b

# Logit lens semantic consistency
python -m interpretability.logit_lens --model gemma2_9b --topk 20

# Phonestheme onset attention
python -m interpretability.attention_analysis --model gemma2_9b

# Causal tracing (ROME-style activation patching)
# ~3 hours on A100 for n_stimuli=20; use 10 for a faster run
python -m interpretability.causal_tracing --model gemma2_9b --n_stimuli 20

# Pythia contamination trajectory (Pile corpus search)
python -m interpretability.contamination_trajectory

# Aggregate interpretability outputs
python -m interpretability.analyze_interpretability
```

All interpretability results are written to `results/interpretability/`.

## Caching

The API client caches all model responses to `cache/` using SHA-256 hashes of
the request parameters. Re-running an experiment will replay from cache rather
than issuing new API calls. Delete `cache/` to force fresh requests.
